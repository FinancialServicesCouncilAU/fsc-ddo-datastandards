Element TMDReport

This element defines the root element of the report

Element ReportExtractionDateTime

Date that TMD report was generatet.

Attribute StandardVersion

Should automatically pick up the Standard's current version

Element TMDs

The collection of TMDs in this report

Element ProductName

Financial instrument (Product) name. Note name is subject to change over time, so recommended don’t rely on this over time

Element IssuerIdentifier

A unique identifier of the Issuer CHECK: this is not present in the Data standard

Element ProductIssuerName

A collection of of alternitive identifiers for the Issuer. CHECK: this is not present in the Data standard

Element DateApproved

Date that this TMD version was approved

Element TMDVersion

The version number of the TMD

Element TMDStatus

The status of the TMD which indicates whether it should be distributed. A status of unavailable or closed will be a signal to distributor to stop retail distribution (unless the distribution is excluded dealing).

Element MandatoryReviewDate

Next mandatory review date for this TMD version. A review can occur before this date.

Element ContactDetails

Mandatory contact details for more information on this TMD

Element TMDCharacteristics

Element ReviewTriggers

List one trigger per subrecord. As distributors don’t need to action review triggers, there is no need for standardisation/ validation of the text

Element AcquisitionOutsideTM

Is there requirement for distributors to report on ALL dealings outside target market?

Element SFCDDOStandardRequired

If AcquisitionOutsideTM is true, should report follow FSC DDO data standard for transactions? Only complete if AcquisitionOutsideTM is true

Element SFCDDOStandardVersionRequired

If AcquisitionOutsideTM is true, should report follow FSC DDO data standard for transactions? Only complete if AcquisitionOutsideTM is true

Element OutsideTMDFrequencyOfReport

Required frequency of report by distributors on dealings outside target market.

Element OutsideTMDReportDeadlineDays

Required deadline for dealings report after end of relevant period (see previous field) – in number of days. Default is 10 days

Element ComplaintsReporting

Required content of complaints report by distributors

Element ComplaintsFrequencyOfReport

Required frequency of report on complaints. Default is at time of complaint.

Element ComplaintsReportDeadlineDays

Required deadline for complaints report after end of relevant period (see previous field) – in number of days. Default is 10 days

Element RequireSignificantDealingsReport

a constant value as a reminder the Significant Dealings is required CHECK: should this include the other common fields ie stanards version etc.

Element Appropriateness

Explanation how issuer considers TMD performs against appropriateness test as required by ASIC RG

Element AvailableDirect

Is product available direct (non intermediated) retail?

Element DirectConditions

Additional conditions on direct (non intermediated) retail. Only complete if AvailableDirect is true.

Element AvailableOnPlatform

Is product available on platforms?

Element PlatformConditions

Additional conditions on distribution to platforms. Only complete if AvailableOnPlatform is true.

Element AvailableAdvised

Is product available on platforms?

Element AdvisedConditions

Additional conditions on distribution to clients subject to personal advice. Only complete if AvailableAdvised is true.

Element InvestmentObjectiveDescription

A description of the investment objective of the product, including growth/ defensive split

Element ProductUseDescription

A description of the investment objective of the product, including asset allocation and diversification information

Element InvestmentTimeframeDescription

A description of the minimum suggested timeframe for holding the product

Element RiskAndReturnDescription

A description of the product risk and return metrics, include risk band and the return hurdle rate

Element RedemptionFrequencyDescription

A description of the intended frequency and mechanism for which investors redeem interests in the product

Element LiquidityDescription

A description of the liquidity of the assets held by the fund

Element Characteristic

The Characteristic to be assessed for Target Market

Element Value

The assessed value of the product against the Characteristic

Enumuration FrequencyOfReportEnum.OnEvent

indicates the repot should happen when the event occurs - ie at point of sale or when complaint raised

Enumuration FrequencyOfReportEnum.EndOfMonth

end of calendar month

Enumuration FrequencyOfReportEnum.EndOfQuarter

end of calendar quarter
